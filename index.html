Road tripping is one of my life happiness activities and I have worked towards various travel goals over the years. My current goal is to visit 125 metropolitan areas across the United States and Canada. This project tracks my progress towards that goal. A full-sized poster of this visualization is pinned to my wall.

The data science skills used in this project include: Writing a decision-making algorithm to recommend cities for the list • Projecting maps, parsing KML files, and similar GIS tasks • Visualizing project results


GitHub repository: road_trips
This project suggests destinations for a US-based aspiring world traveler, based on multiple "point of interest" and "ease of travel" factors. For each country, the project suggests up to three cities to visit and lists nearby UNESCO world heritage sites.

Data science skills: Pulling data from servers via API Requests • Fusing data streams from eight sources, despite lack of a standardized merge key • Shaping and refining a composite dataset • Building a multi-criteria decision-making algorithm to recommend destinations • Visualizing project results

GitHub repository: travel_suggestions
IN PROGRESS: The states of the United States have unequal populations, unequal land area and borders that cut through major urban centers. This project creates an interactive tool for exploring how targeted changes to state borders could produce better state borders.

Data science skills: • Conducting GIS analysis across multiple levels of geography • Creating an engaging educational tool so that a public audience can understand a public policy issue better • Building an interactive web-based data exploration tool (shiny-based) • Visualizing data

GitHub repository: better_state_borders
IN PROGRESS: Scatterplots can become ineffective or misleading if there are too many data points because most points are invisible beneath other nearby points. This project discusses solutions to this problem and illustrates them with a map of the US population.

Data science skills: Processing data in parallel • Applying MapReduce "big data" paradigm programming (though the data is not that big) • Projecting maps and similar GIS tasks • Pulling data via API and non-API ("web scrape") server data pulls • Applying clustering algorithms • Visualizing project results

GitHub repository: crowded_scatterplot
This project creates an interactive tool for estimating the chances that the weather will be temperate in a specific part of the contiguous United States in in a given month and time range. While envisioned as a road tripping tool, users can specify any desired temperature range for any time range.

Data science skills: Pulling data via API and non-API ("web scrape") server data pulls • Conducting robust, punctuated data pulls • Generating Voronoi decompositions and performing similar GIS interpolation tasks • Building an interactive data tool using RShiny • Visualizing data interactively
